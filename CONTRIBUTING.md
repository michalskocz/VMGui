# Contributing to VMGui

Thank you for your interest in contributing to **VMGui**! 🎉  
We welcome all kinds of contributions, from bug reports to new features.

---

## 📌 How You Can Contribute

There are many ways to help improve the project:

- 🐛 Reporting bugs (Issues)
- 💡 Suggesting new features
- 🛠️ Fixing bugs
- ✨ Adding new functionality
- 📖 Improving documentation

---

## 🚀 Getting Started

1. **Fork the repository**
2. **Clone your fork**
    ```bash
    git clone https://github.com/YOUR_USERNAME/VMGui.git
    ````

3. Navigate to the project directory:

    ```bash
    cd VMGui
    ```
4. Create a new branch:

    ```bash
    git checkout -b feature/your-feature-name
    ```

---

## 🧑‍💻 Coding Guidelines

To keep the codebase consistent:

* Use clear and descriptive names
* Follow the existing code style
* Add comments where necessary
* Avoid unnecessary complexity and duplication

If the project uses a linter or formatter, make sure your code passes it.

---

## 🧪 Testing

* Ensure the project builds successfully
* Add tests if you introduce new functionality (if applicable)
* Do not break existing features

---

## 🐛 Reporting Bugs

When creating an issue, please include:

* A clear description of the problem
* Steps to reproduce
* Expected behavior
* Actual behavior
* Logs or screenshots (if available)

---

## 💡 Suggesting Changes

Before working on large changes:

1. Open an issue
2. Describe your idea
3. Wait for feedback

This helps avoid unnecessary work.

---

## 🔁 Pull Requests

1. Make sure your branch is up to date:

   ```bash
   git pull origin main
   ```

2. Commit your changes:

   ```bash
   git commit -m "feat: add new feature X"
   ```

3. Push to your fork:

   ```bash
   git push origin feature/your-feature-name
   ```

4. Open a Pull Request

---

## 📝 Commit Message Convention

We recommend using:

* `feat:` – new feature
* `fix:` – bug fix
* `docs:` – documentation
* `refactor:` – code refactoring
* `test:` – tests

Example:

```
feat: add settings panel
fix: resolve crash on startup
```
