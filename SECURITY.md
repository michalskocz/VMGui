# Security Policy

## 🔐 Supported Versions

The following versions of **VMGui** are currently supported with security updates:

| Version        | Supported |
|----------------|-----------|
| 1.0            | ✅         |
| Older versions | N/A       |

Please make sure you are using the latest version before reporting a vulnerability.

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead, report it responsibly:

- 📧 Email: michal.skoczylas14@gmail.com


Include as much detail as possible:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

---

## ⏱️ Response Time

We aim to:

- Acknowledge your report within **72 hours**
- Provide an initial assessment within **3–5 days**
- Release a fix as soon as possible

---

## 🔒 Disclosure Policy

- Please **do not publicly disclose** the vulnerability until it has been addressed
- Once fixed, we may publish a security advisory
- You will be credited (unless you prefer to remain anonymous)

---

## 🛡️ Security Best Practices

When using VMGui:

- Always download from trusted sources
- Keep the application up to date
- Avoid running untrusted virtual machine configurations
- Report suspicious behavior

---

## ⚠️ Scope

This policy applies to:

- The core VMGui application
- Official releases and builds

It does **not** cover:

- Third-party plugins or modifications
- Unsupported or outdated versions

---

## ❤️ Thank You

We appreciate responsible disclosure and your help in keeping VMGui secure 🙌
